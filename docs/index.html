它只在该打扰你的时候打扰你

三条原则，每一条都对应源码里一段明确的判定逻辑。

高优先级

需要你介入，才尽快通知

权限确认、等待输入、填写表单、真正卡住——这些你不在就卡住的场景，短延迟重检确认你已离开后立刻推送。

反误报

不把过程态当成终态

不再把每一次 Stop 都当「任务完成」，也不把所有工具失败都当「需要你处理」。优先保证「需要你回来」的通知准时到达。

不打扰

你盯着终端，它就闭嘴

检测前台应用与精确窗口匹配——只要你正看着这个终端会话，即使有事件也静默不发，避免重复打扰。

通知优先级模型

不是一触发就发。每一档都有自己的「发送条件」，这才是它最反直觉、也最有用的设计。

高 需要你立即介入
  • PermissionRequest权限确认
  • Notification(idle_prompt)等待你补充输入
  • Elicitation填表单 / 登录
  • StopFailure认证 / 计费 / 真正卡住
发送条件

5 秒后重检，确认你已离开终端或锁屏，再推送。你还在看 → 不打扰。

中 任务完成，但你不在
  • TaskCompleted任务完成（多 agent 工作流）
发送条件

锁屏，或切到非终端 / 编辑器应用时才发。你正盯着结果 → 不打扰。

低 过程态，能省则省
  • SessionEnd会话结束
  • Stop本轮响应结束
发送条件

仅在你离开时发，其他情况一律不发。把噪音降到最低。

它怎么知道你「正在看终端」

一组决策逻辑，加上三项 macOS 原生检测能力。

🔒

锁屏检测

调用 macOS Quartz 的 CGSessionCopyActive，准确判断屏幕是否锁定。锁屏即视为「你不在」，立即放行通知。

▣

前台应用检测

通过 AppleScript 读前台进程，按「路径 > Bundle ID > 进程名」三重匹配，识别你是不是在终端或编辑器里。

▢

精确窗口匹配

iTerm2 / Terminal 按 tty 与 unique id 匹配会话窗口，避免 tmux、多窗口场景下误判「你在看」。

一个通知系统，接住所有 AI 工具

不同工具用各自的官方机制接入，精度各有不同——但统一汇入同一套优先级模型。

Claude Code 精度最高

官方 hooks JSON

  • PermissionRequest
  • Notification(idle_prompt)
  • Elicitation · StopFailure
  • TaskCompleted · SessionEnd
Cursor 轻量

官方 hooks JSON

  • stop

OpenCode 插件机制

官方插件

  • session.error
  • session.idle
Codex best-effort

实验性 hooks

  • PostToolUse(Bash)
  • Stop（启发式）

事件面较窄，按「是否在等你回复」做启发式识别。

通用 CLI 入口
smart-notify.sh --source my-tool --event approval_required --kind intervention --priority high

未来接入新工具，只需把它的事件映射成这套统一参数。

安装与配置

macOS 上一条命令搞定，安装脚本会自动检测你装了哪些 AI 工具。

  1. 1

    一键安装（推荐）

    curl -fsSL https://raw.githubusercontent.com/Michaelooo/cc-notify/main/install.sh | bash

    或本地安装：git clone 后运行 ./install.sh。脚本会交互式让你勾选要接入的工具。

  2. 2

    配置 Bark

    iPhone 安装 Bark，复制你的 Key。安装时按提示输入，或预先 export BARK_KEY=你的Key。

  3. 3

    授权辅助功能

    首次使用需在 系统设置 → 隐私与安全 → 辅助功能 中授权你的终端应用——这是判断「你是否正在看终端」的前提。

  4. 4

    测试一下

    ~/.cc-notify/bin/smart-notify.sh "测试" "安装成功" "normal"

    手机收到推送即配置完成。配置文件统一在 ~/.cc-notify/，升级时自动备份并保留你的自定义 hooks。
