# AI Coding 场景通知功能深度调研报告

## 一、场景定义与需求分析

```
┌─────────────────────────────────────────────────────────────┐
│                      AI Coding 场景                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   场景 1: AI 需要确认选择                                     │
│   ┌─────┐    "使用 A 方案还是 B 方案？"    ┌─────┐          │
│   │ AI  │ ───────────────────────────────▶ │用户 │          │
│   │编码中│    用户切屏中，需要被通知          │切屏 │          │
│   └─────┘                                  └─────┘          │
│                                                              │
│   场景 2: 任务完成/失败通知                                   │
│   ┌─────┐    "代码生成完成！"              ┌─────┐          │
│   │ AI  │ ───────────────────────────────▶ │用户 │          │
│   │执行中│    或 "API Key 错误，请检查"      │不在 │          │
│   └─────┘                                  └─────┘          │
│                                                              │
│   场景 3: 需要信息补充                                        │
│   ┌─────┐    "请提供更多上下文信息..."       ┌─────┐          │
│   │ AI  │ ───────────────────────────────▶ │用户 │          │
│   │等待中│                                │切屏 │          │
│   └─────┘                                  └─────┘          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 核心需求
| 需求维度 | 描述 |
|---------|------|
| **即时性** | 用户切屏后能被及时提醒，延迟 < 5 秒 |
| **可达性** | 即使手机锁屏，通知也能到达 |
| **多终端** | 桌面 → 手机 → 手表 全链路覆盖 |
| **低打扰** | 可配置通知级别，避免过多干扰 |
| **国内可用** | 无需特殊网络环境，服务稳定 |

## 三、各方案深度解析

### 3.1 首选推荐：Bark（轻量级推送）

**为什么首选 Bark：**
1. ✅ **完全免费**，无调用限制
2. ✅ **开箱即用**，无需注册开发者账号
3. ✅ **支持 Apple Watch**（通过 APNs 自动同步）
4. ✅ **可自建服务器**，数据可控
5. ✅ **国内开发**，网络无阻碍

**使用方式：**

```bash
# 1. App Store 搜索 "Bark" 安装
# 2. 打开 App 获取你的 Key（如: xxxxxxxx）

# 3. 发送通知（最简单方式）
curl "https://api.day.app/YOUR_KEY/标题/内容"

# 4. 高级用法（带分组、声音、图标）
curl "https://api.day.app/YOUR_KEY/AI任务完成/代码生成成功?group=aicoding&sound=alarm"
```

**Python 集成：**
```python
import requests

def notify_bark(title, body, key="YOUR_KEY"):
    """AI Coding 完成通知"""
    url = f"https://api.day.app/{key}/{title}/{body}"
    # 或者使用 POST 更灵活
    url = f"https://api.day.app/{key}"
    data = {
        "title": title,
        "body": body,
        "group": "aicoding",
        "sound": "glass",  # 提示音
        "level": "timeSensitive"  # 时效性通知（突破专注模式）
    }
    requests.post(url, json=data)

# 使用示例
notify_bark("Claude 需要确认", "选择 React 还是 Vue 框架？")
```

**自建服务器（可选）：**
```yaml
# docker-compose.yml
version: "3"
services:
  bark:
    image: finab/bark-server:latest
    ports:
      - "8080:8080"
    volumes:
      - ./data:/data
    restart: unless-stopped
```

### 3.4 终端原生通知（macOS）

**适用场景：** 用户在同一台 Mac 上工作，只是切换了窗口

```bash
# 方式 1：系统自带 osascript
osascript -e 'display notification "AI 需要您确认" with title "AI Coding" sound name "Glass"'

# 方式 2：terminal-notifier（更强大）
brew install terminal-notifier

terminal-notifier \
  -title "AI Coding" \
  -subtitle "任务完成" \
  -message "代码已生成，请查看" \
  -sound "default" \
  -appIcon "https://cursor.sh/favicon.ico" \
  -execute "open -a Cursor"

# 方式 3：OSC 转义序列（终端内）
echo -e "\033]777;notify;AI Coding;任务完成\007"
```
